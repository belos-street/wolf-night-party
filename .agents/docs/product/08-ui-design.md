# 狼人杀 - 本地联机版 界面设计

## 1. 页面列表

| 页面 | 说明 | 触发条件 |
|------|------|---------|
| 加入页面 | 输入昵称加入游戏 | 访问网站时 |
| 等待大厅 | 显示已加入玩家列表 | 加入游戏后 |
| 角色查看 | 显示分配的角色 | 游戏开始后 |
| 夜晚操作 | 各角色夜间操作界面 | 夜晚阶段 |
| 夜晚等待 | 非当前行动玩家的等待界面 | 夜晚阶段 |
| 白天通报 | 显示昨夜死亡信息 | 白天阶段 |
| 白天投票 | 投票放逐界面 | 白天阶段 |
| 游戏结束 | 显示胜利阵营和角色 | 游戏结束时 |
| 断线页面 | 显示断线重连提示 | WebSocket 断开时 |
| 无法加入页面 | 游戏进行中无法加入 | 游戏开始后访问 |
| 帮助弹窗（全局） | 显示规则摘要与职业说明 | 点击页面右上角“❓帮助” |

## 3. Modal 组件说明

游戏中使用 Modal（模态弹窗）来处理轻量级的交互提示，避免频繁跳转页面打断游戏节奏。

> **全局帮助按钮规则**：
> - 游戏进行中所有页面右上角都显示 `❓帮助` 按钮（不影响当前阶段流程）
> - 点击后弹出“规则帮助”Modal，展示规则摘要与职业说明
> - 帮助内容为本地静态文案，不依赖服务端事件

### 3.1 遗言提示 Modal
```
┌─────────────────────────────────┐
│                                 │
│      ┌─────────────────────┐   │
│      │  🎤 遗 言            │   │
│      │                     │   │
│      │  请死亡玩家发表遗言   │   │
│      │                     │   │
│      │  李四、王五           │   │
│      │  （仅显示昵称，不显示身份）│   │
│      │                     │   │
│      │  ┌───────────────┐ │   │
│      │  │  遗 言 结 束   │ │   │
│      │  └───────────────┘ │   │
│      └─────────────────────┘   │
│                                 │
└─────────────────────────────────┘
```
- **触发时机**: 死亡通报后（猎人开枪后或无猎人开枪时）
- **显示内容**: 提示当前需要发表遗言的死亡玩家昵称（不显示身份）
- **操作**: 死亡玩家线下发言后，点击"遗言结束"继续流程
- **注意**: 女巫毒杀和被猎人带走的玩家无遗言；遗言是线下社交环节，系统仅做提示，不记录内容

## 4. 主机面板设计

### 4.1 主机面板（等待阶段）
```
┌─────────────────────────────────┐
│  🐺 主机控制面板                 │
│─────────────────────────────────│
│  当前玩家: 6/7                   │
│  张三, 李四, 王五, 赵六,         │
│  孙七, 周八                      │
│─────────────────────────────────│
│                                 │
│  标准模式（6-12人）:             │
│  ┌───────────────────────────┐ │
│  │ 当前预设: 7人局（只读）      │ │
│  │ 狼人: 2                    │ │
│  │ 平民: 3                    │ │
│  │ 预言家: 1                  │ │
│  │ 女巫: 1                    │ │
│  │ 猎人: 0                    │ │
│  │ 守卫: 0                    │ │
│  │ 白痴: 0                    │ │
│  └───────────────────────────┘ │
│                                 │
│  人数变化时自动切换预设           │
│                                 │
│  ┌───────────────────────────┐ │
│  │       开 始 游 戏          │ │
│  └───────────────────────────┘ │
│                                 │
└─────────────────────────────────┘
```

**功能说明**:
- **固定预设**: 根据当前人数自动加载标准平衡配置，主机不可手动调整
- **只读展示**: 角色面板仅展示预设结果，不提供 `+/-` 按钮
- **人数门槛**: 仅支持 6-12 人开局，超出范围时"开始游戏"按钮禁用
- **玩家列表**: 显示已加入的玩家昵称

### 4.2 主机面板（游戏进行中）
```
┌─────────────────────────────────┐
│  🐺 主机控制面板                 │
│─────────────────────────────────│
│  状态: 🌙 夜晚 - 狼人刀人        │
│  第 2 天                        │
│─────────────────────────────────│
│                                 │
│  存活玩家: 6                     │
│  死亡玩家: 1                     │
│  ────────────────────────────  │
│  💀 李四（狼人刀杀）             │
│                                 │
│  重新开始：本局进行中不可用       │
│                                 │
└─────────────────────────────────┘
```

**功能说明**:
- **状态显示**: 当前游戏阶段（夜晚/白天）+ 子阶段（狼人刀人/预言家查验等）
- **天数显示**: 当前第几天
- **存活/死亡统计**: 实时显示存活和死亡玩家数量
- **死亡记录**: 显示已死亡玩家及死因
- **重新开始**: 仅在 `WAITING` 或 `ENDED` 状态可用，游戏进行中不可用（避免人为中断影响公平性）

### 4.3 主机面板（角色分配后）
```
┌─────────────────────────────────┐
│  🐺 主机控制面板                 │
│─────────────────────────────────│
│  状态: 🎭 角色已分配             │
│  等待玩家查看角色...             │
│─────────────────────────────────│
│                                 │
│  角色分配完成                    │
│  所有玩家已收到自己的角色信息      │
│                                 │
│  即将自动进入夜晚阶段...          │
│                                 │
└─────────────────────────────────┘
```

**功能说明**:
- **状态提示**: 显示角色分配已完成，等待玩家查看
- **自动流转**: 角色分配完成后自动进入夜晚阶段，无需主机确认
- **公平性保证**: 主机与其他玩家一样，只能看到自己的角色，无法查看其他玩家身份
