# 运行与部署栈

## 1. 部署目标

实现“主机手机一键开局”：

- 手机运行 Termux 中的后端服务
- 手机开启 WiFi 热点
- 其他玩家连接热点后通过浏览器访问主机地址加入

## 2. 运行环境

| 项 | 方案 |
|----|------|
| 主机端 | Android + Termux |
| 服务运行时 | Node.js >= 22.13.0（推荐 22.x LTS） |
| 包管理器 | `pnpm`（建议） |
| 客户端 | 手机/平板/电脑浏览器 |
| 网络 | 同一局域网（热点） |

## 3. 建议目录与脚本

```text
project/
  server/
  client/
  scripts/
    dev-server.sh        # 开发运行
    prod-server.sh       # 生产运行（Termux）
```

建议命令（示例）：

- 开发：`pnpm dev`
- 生产：`pnpm build && pnpm start`

## 4. 网络与端口建议

- 默认端口 `3000`
- 启动时打印本机可访问地址（如 `192.168.43.1:3000`）
- 客户端只需要输入 `IP:PORT` 即可访问
- WebSocket 与 HTTP 共用同一服务端口，减少热点环境下的配置复杂度

## 5. 可观测与容错建议

- 启动日志：版本、端口、局域网地址
- 关键事件日志：加入、开始、阶段切换、胜负判定、断线重连
- 异常保护：主进程异常退出时给出清晰提示（需重启开局）
- 连接健康检查：WS 心跳（ping/pong）+ 超时回收

## 6. 非目标（第一版不做）

- 云端部署
- 持久化数据库
- 跨公网联机
- 多房间并行

## 7. 版本基线建议

1. Node：`>=22.13.0`
2. npm：随 Node 22 默认版本
3. pnpm：`>=9`
4. 首次部署前执行：
   - `node -v`
   - `pnpm -v`
   - `pnpm install`
