# 测试策略

## 1. 目标

优先保障“规则正确性 + 状态机稳定性 + 联调可回归”。

## 2. 测试分层

### 2.1 后端单元测试（优先级最高）

- 工具：`node:test`（Fastify 脚手架默认）+ `c8` 覆盖率
- 覆盖对象：
  - 夜晚结算顺序（狼刀/守卫/女巫）
  - 投票与重投
  - 白痴翻牌与失票权
  - 猎人开枪触发时机
  - 胜利条件（6-9 屠城、10-12 屠边）

### 2.2 后端集成测试

- 覆盖 `HTTP + WebSocket` 主流程：
  - `join -> start -> 夜晚 -> 白天 -> 结束`
  - 断线与重连
  - 非法事件被拒绝（阶段不匹配、身份不匹配）

### 2.3 前端测试

- 组件测试（可选）：帮助 Modal、投票禁用态、阶段提示
- 前端测试框架可后续补充（推荐 `Vitest + Testing Library`）
- E2E（第二阶段）：多端加入与关键流程回归

## 3. 首版覆盖门槛（建议）

1. `game/engine` 核心逻辑行覆盖率 >= 85%
2. 协议校验与错误码分支覆盖率 >= 80%
3. 至少 1 条全流程集成用例可稳定通过

## 4. 测试数据策略

1. 固定随机种子（确保角色分配可复现）
2. 使用工厂函数构建最小测试房间（6 人基线）
3. 一条规则一个断言主题，避免“巨型测试”

## 5. 建议脚本

```bash
# server
pnpm test
pnpm test:watch
pnpm test:coverage

# client
pnpm test
```

## 6. 回归清单（每次发版前）

1. 6 人局完整跑通
2. 12 人局完整跑通
3. 平票重投分支
4. 白痴翻牌分支
5. 猎人狼刀触发与放逐触发两分支
6. 断线 2 分钟超时判死分支
