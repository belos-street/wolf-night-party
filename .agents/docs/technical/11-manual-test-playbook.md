# 人工联调测试手册（多标签页版）

## 1. 目标

- 在本地手动验证首版「可开局、可推进、可结算、可重连」。
- 使用同一浏览器多个标签页模拟多玩家，不依赖多台设备。
- 输出可复用的测试记录，支持后续回归。

## 2. 测试环境

- OS：Windows / macOS / Linux（任意）
- Node：`>=22.13.0`
- 包管理：`pnpm`
- 浏览器：Chrome / Edge（推荐）
- 项目根目录：`wolf-night-party`

## 3. 启动步骤

1. 安装依赖（首次）：
```bash
pnpm install
```
2. 启动前后端：
```bash
pnpm dev
```
3. 打开前端地址（默认）：
`http://127.0.0.1:5173`

## 4. 多标签页建房方法（6人最小局）

### 4.1 标签页规划

- T1：Host（主机）
- T2：P2
- T3：P3
- T4：P4
- T5：P5
- T6：P6

### 4.2 加入顺序

1. 在 6 个标签页分别输入不同昵称并加入。
2. 预期：大厅内显示 6 名玩家，T1 为主机。
3. 预期：只有 T1 可点击「开始游戏」。

## 5. 主流程测试（P0）

## 5.1 开局与身份确认

1. T1 点击开始游戏。
2. 所有标签进入身份页（`ROLE_VIEW`）。
3. 所有标签点击确认身份。
4. 预期：进入夜晚阶段（`NIGHT_*`）。

## 5.2 夜晚阶段推进

1. 按阶段执行动作（狼人/预言家/守卫/女巫），其余玩家观察不可操作态。
2. 若当阶段无可执行角色，可等待或触发默认推进。
3. 预期：夜晚结束后进入白天揭示（`DAY_REVEAL`）。

## 5.3 白天阶段推进

1. 依次推进：揭示 -> 遗言 -> 讨论 -> 投票。
2. 在投票阶段，所有存活可投票玩家提交投票。
3. 预期：进入投票结果（`DAY_VOTE_RESULT`），并展示结果信息。
4. 继续推进到下一轮夜晚，直至游戏结束（`ENDED`）。

## 6. 异常与恢复测试（P0）

## 6.1 刷新重连（sessionStorage）

1. 任一玩家标签页刷新（F5）。
2. 预期：自动恢复该玩家会话并重连，不会变成新玩家。
3. 预期：座位号和玩家身份保持不变。

## 6.2 断线后 120 秒内重连

1. 关闭任一非主机标签页（如 T4）。
2. 其他标签应看到断线倒计时提示。
3. 在 120 秒内重新打开该标签页并访问前端地址。
4. 预期：玩家恢复连接，倒计时消失，可继续游戏。

## 6.3 断线超时判死

1. 关闭任一存活玩家标签页。
2. 等待 120 秒不重连。
3. 预期：该玩家被判死并广播结果，流程继续（或触发胜负结算）。

## 6.4 人数边界校验

1. 仅 5 人加入时尝试开局。
2. 预期：开局被拒绝，提示标准模式仅支持 `6-12` 人。

## 7. 分支规则测试（建议至少覆盖一次）

- 平票 -> 重投 -> 再平票路径。
- 白痴被投出后的翻牌与失去投票权路径。
- 猎人触发技能路径（被票出 / 夜间被刀）至少一种。

## 8. 通过标准（建议）

- 无阻断级错误（无法开局、阶段卡死、无法结算）。
- 关键异常可恢复（刷新重连、断线重连）。
- 规则分支与文档预期一致。

## 10. 执行建议

- 优先跑 6 人局（最快闭环），再跑 12 人局（覆盖复杂配置）。
- 每次修复后至少回归 TC-01 + TC-02 + TC-03。
- 若阶段卡住，记录当前 `phase`、玩家列表、最后一次操作事件名。
